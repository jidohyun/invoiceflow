## 1. 서비스 개요

### 가. 배경과 개발목표

#### 1) 프리랜서 시장의 성장과 미수금 문제

코로나 이후 프리랜서 시장은 전 세계적으로 폭발적으로 성장하고 있다. 미국에서는 노동 인구의 36%가 프리랜서로 일하고, 한국에서도 N잡러, 사이드 프로젝트, 1인 기업이 주류로 떠오르고 있다. 그러나 프리랜서들이 가장 큰 어려움을 겪는 부분은 "받기로 한 돈을 제때 받는 것"이다. Freshbooks의 2023년 조사에 따르면 프리랜서의 74%가 클라이언트의 늦은 결제 때문에 스트레스를 받는다고 답했다. 평균적으로 프리랜서가 송장 발송 후 결제를 받기까지 걸리는 시간은 30~45일이고, 이 중 약 40%는 due_date를 넘긴다. 단순히 "기억하고 독촉 메일을 보내는 것"이 본업보다 더 큰 부담이 되는 경우가 많다.

#### 2) 기존 도구의 한계

FreshBooks, Wave, QuickBooks 같은 송장 관리 도구는 이미 시장에 존재한다. 하지만 이들은 대부분 미국/유럽 시장에 최적화되어 있고, 한국 프리랜서가 사용하기에는 통화 처리, 세금계산서 발행, 한국어 UX 등에서 부족하다. 또한 자동 리마인더 기능이 있긴 하지만 단순히 "보낸 후 N일 후에 메일을 보낸다" 수준이고, 클라이언트별 결제 패턴 분석이나 현금 흐름 예측 같은 인텔리전트 기능은 부족하다. AutoMyInvoice는 이 갭을 메우기 위해 기획되었다. 송장 발행부터 결제 수금, 자동 리마인더, 연체 감지, 클라이언트 분석까지 한 번에 처리하는 올인원 SaaS다.

### 나. 서비스 개념

AutoMyInvoice는 프리랜서와 1인 기업을 위한 자동 송장 관리 SaaS다. 사용자가 송장을 발행하면 자동으로 클라이언트에게 이메일을 발송하고, due_date가 지나도 입금이 되지 않으면 자동으로 3단계 리마인더를 발송한다. 결제는 Paddle을 통한 카드 결제 링크를 자동 생성해서 클라이언트가 한 번의 클릭으로 결제할 수 있게 한다. 사용자에게 제공하는 가치는 다음과 같다.

- **시간 절약:** 송장 발송, 독촉, 미수금 추적에 들어가는 시간을 0에 가깝게 줄여준다.
- **수금률 개선:** 자동 리마인더와 연체 감지로 미수금을 줄이고, 평균 결제 소요일을 단축한다.
- **데이터 인사이트:** 클라이언트별 결제 패턴, 향후 30/60/90일 캐시플로우 예측, 리마인더 효과 분석을 제공한다.
- **모바일 우선:** Android와 iOS 네이티브 앱에서도 모든 기능을 사용할 수 있다.

### 다. 주요 타겟 고객

#### 1) 1차 타겟: 월 5~20건 송장을 발행하는 프리랜서

IT 개발자, 디자이너, 마케터, 컨설턴트 등 본업에 집중하고 싶지만 송장 관리에 시간을 빼앗기는 프리랜서가 1차 타겟이다. 이들은 보통 엑셀이나 노션으로 송장을 관리하고, 이메일로 직접 발송한다. 미수금이 생기면 직접 리마인더를 보내는데, 이 과정에서 클라이언트 관계가 어색해지거나 잊어버려서 영영 못 받는 경우도 있다.

#### 2) 2차 타겟: 직원 1~5명의 스튜디오/에이전시

작은 디자인 스튜디오, 컨설팅 펌, 광고 에이전시처럼 여러 클라이언트를 동시에 관리하는 소규모 사업자가 2차 타겟이다. 이들은 더 많은 송장을 더 빠르게 처리해야 하고, 클라이언트별로 다른 결제 조건과 리마인더 정책이 필요하다.

### 라. 주요 서비스 내용

#### 1) 주요 서비스 기능

**① 송장 자동 생성 및 발송**
사용자가 클라이언트, 금액, 통화, due_date를 입력하면 INV-YYYYMM-XXXX 형식으로 자동 채번된 송장이 생성된다. 발송 버튼 한 번으로 클라이언트에게 송장 이메일이 자동 발송되고, Paddle 결제 링크가 함께 포함된다.

**② 자동 3단계 리마인더**
송장 발송 후 D+1(친절한 리마인더), D+7(팔로우업), D+14(최종 통보) 3단계 리마인더가 자동으로 스케줄링된다. 각 리마인더는 클라이언트의 timezone에 맞춰 평일 오전 9-10시 사이 랜덤 시간에 발송되어 자연스러움을 더한다.

**③ 연체 자동 감지 & 알림**
매일 오전 9시 UTC에 OverdueScanner가 실행되어 due_date가 지난 송장을 자동으로 overdue 상태로 전환하고, 별도의 OverdueNotificationWorker가 사용자에게 연체 알림 이메일을 발송한다.

**④ 부분결제 추적**
클라이언트가 일부만 결제한 경우 record_payment API를 통해 부분 결제를 기록할 수 있다. paid_amount가 누적되어 invoice.amount에 도달하면 자동으로 paid 상태로 전환된다. DB 레벨 atomic operation으로 동시성 경쟁을 방지한다.

**⑤ 결제 자동 수금 (Paddle Webhook)**
Paddle에서 결제가 완료되면 webhook을 통해 자동으로 송장이 paid 상태로 전환되고, pending 리마인더가 취소된다. Idempotent 처리로 중복 webhook도 안전하게 처리한다.

**⑥ 분석 대시보드**
월별 수금 추이, 송장 상태별 분포, 미결제 송장 aging(0-30일/31-60일/61-90일/90일+), 캐시플로우 예측(향후 90일 수금 예상)을 차트로 시각화한다. Chart.js와 Phoenix LiveView phx-hook으로 구현했다.

**⑦ 클라이언트 결제 패턴 분석**
클라이언트별 평균 결제 소요일(avg_payment_days), on-time rate(due_date + 3일 이내 결제율), 클라이언트 랭킹을 제공한다. 어떤 클라이언트가 잘 갚고 어떤 클라이언트가 자주 늦는지 한눈에 파악할 수 있다.

**⑧ 리마인더 효과 분석**
3단계 리마인더의 step별 open rate, click rate, 리마인더 발송 후 결제까지 평균 소요일을 분석한다. 어느 리마인더가 가장 효과적인지 데이터로 확인할 수 있다.

**⑨ REST API**
모든 기능에 대한 REST API를 제공해서 모바일 앱(Android, iOS)에서도 동일한 기능을 사용할 수 있다. Bearer token 인증, ApiAuth plug, 사용자 scoping 등 보안 패턴을 따른다.

## 2. 서비스 경쟁력 / 차별화

### 가. 경쟁(유사) 서비스 현황

| 서비스 | 강점 | 약점 |
|---|---|---|
| FreshBooks | 글로벌 1위, 견고한 기능 | 한국어 미지원, 비싼 가격 (월 $19~) |
| Wave | 무료, 깔끔한 UX | 미국/캐나다 외 결제 제한, 한국 미지원 |
| QuickBooks | 회계 기능 통합 | 복잡함, 학습 곡선 가파름 |
| Stripe Invoicing | Stripe 결제 통합 | 분석/리마인더 기능 약함, 한국 결제 부분 지원 |

#### 1. FreshBooks
FreshBooks는 캐나다에서 시작한 글로벌 송장 관리 SaaS다. 1,000만 명 이상의 사용자를 보유하고 있고, 송장 발행, 리마인더, 시간 추적, 회계 기능까지 통합 제공한다. 다만 한국어 UI를 지원하지 않고, 가장 저렴한 플랜이 월 $19로 한국 프리랜서에게는 비싸게 느껴질 수 있다.

##### 가. 장점
1) **풍부한 기능 세트:** 송장, 시간 추적, 경비, 회계, 프로젝트 관리까지 한 번에 처리한다.
2) **검증된 안정성:** 10년 이상 운영되어 결제 흐름과 리마인더 시스템이 안정적이다.

##### 나. 단점
1) **한국 시장 미지원:** 한국어 UI 없음, 원화 표시는 가능하지만 한국 결제 게이트웨이 미연동.
2) **비싼 가격:** 월 $19부터 시작하는 가격은 1년차 프리랜서에게 부담이다.

#### 2. Wave
Wave는 캐나다의 무료 회계/송장 SaaS다. 송장 발행과 회계 기능을 무료로 제공하고, 결제 처리에서만 수수료를 받는 비즈니스 모델이다.

##### 가. 장점
1) **완전 무료:** 송장 발행, 회계 기능을 모두 무료로 제공해서 비용 부담이 없다.
2) **깔끔한 UX:** 디자인이 모던하고 사용하기 쉽다.

##### 나. 단점
1) **결제 지역 제한:** 미국, 캐나다 외 지역에서는 결제 처리가 제한적이다. 한국에서는 결제 링크를 만들 수 없다.
2) **한국어 미지원:** UI가 영어 전용이라 한국 사용자에게 진입 장벽이 있다.

#### 1. FreshBooks 대비 차별화
AutoMyInvoice는 한국 프리랜서 시장에 최적화되어 있다. 한국어 UI는 물론, 한국 전자세금계산서 발행 자동화(Sprint 4 예정), 원화/달러/유로 등 다중 통화를 자연스럽게 지원한다. 또한 가격이 FreshBooks의 절반 이하로 1년차 프리랜서도 부담 없이 시작할 수 있다.

#### 2. Wave 대비 차별화
Wave는 무료지만 한국 결제 처리가 안 된다. AutoMyInvoice는 Paddle을 통해 한국 카드 결제를 지원하고, 자동 리마인더와 분석 기능을 더 강력하게 제공한다. Wave는 단순 송장 발행 도구지만 AutoMyInvoice는 "수금 자동화 엔진"에 가깝다.

## 3. 개발 계획

### 가. 개발 팀 구성

#### 1) 백엔드 (1명)
**지도현** — Phoenix/Elixir 백엔드, REST API, Oban 백그라운드 잡, 데이터베이스 설계, 분석 엔진 구현. PM 역할 겸직.

#### 2) 프론트엔드 LiveView (지도현 겸직)
Phoenix LiveView 기반 웹 대시보드, Chart.js 차트, Tailwind CSS + DaisyUI 디자인 시스템.

#### 3) 모바일 (지도현 겸직)
Android (Kotlin + Jetpack Compose + Hilt + MVI), iOS (Swift + SwiftUI + Factory + MVI). 단일 OpenAPI spec에서 양 플랫폼 클라이언트 자동 생성.

### 나. 프로젝트 수행 방법

#### 1) 기획 및 디자인
요구사항을 해소할 기능을 기획한다. `/office-hours` 스킬로 YC 스타일의 demand reality 검증을 거치고, 디자인 문서를 작성한다. 디자인 설계 과정에서 백엔드, 웹, 모바일 모두에서 구현 가능한지 고려한다.

#### 2) API 명세서 작성
`packages/api-spec/openapi.yaml`에서 OpenAPI 3.1 spec을 작성한다. 명세서에는 API path, request, response, 에러 코드가 포함된다. 이 spec을 기반으로 모바일 클라이언트가 자동 생성된다.

#### 3) 백엔드 개발
Phoenix Context 패턴을 따른다. 비즈니스 로직은 `lib/auto_my_invoice/`의 Context 모듈에 위치하고, 컨트롤러는 Context 함수만 호출한다. 모든 변경은 Ecto Changeset으로 검증한다. 80% 이상 테스트 커버리지를 유지한다.

#### 4) 모바일 개발
Android는 Jetpack Compose + Hilt + MVI 패턴, iOS는 SwiftUI + Factory + MVI 패턴으로 개발한다. 양 플랫폼 모두 동일한 UX를 제공한다. OpenAPI spec에서 자동 생성된 클라이언트를 사용한다.

#### 5) 코드 리뷰 & 배포
모든 PR은 `/plan-eng-review` 또는 `/review` 스킬로 사전 리뷰를 거친다. CI에서 `mix precommit` (compile + format + test)이 통과해야 머지 가능하다. main 브랜치에 머지되면 Fly.io로 자동 배포된다.

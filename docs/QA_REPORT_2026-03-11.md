## 3. 발견된 버그

### BUG-001: 리마인더 스케줄링 실패 (Timezone 처리 오류)

- **심각도**: 🔴 HIGH
- **상태**: Open
- **위치**: `lib/auto_my_invoice/reminders.ex` → `calculate_send_time/3`
- **재현 경로**: 인보이스 생성 → Send → 리마인더 타임라인 미표시

**증상**:
```
(CaseClauseError) no case clause matching: {:error, :time_zone_not_found}
```

**영향**:
- Send cascade에서 `mark_as_sent` 성공 후 `schedule_reminders` 실패
- 인보이스 상태는 "sent"로 변경되지만 리마인더 0개 생성
- **핵심 비즈니스 로직(자동 리마인더) 완전 파손**

**근본 원인**:
- timezone 라이브러리(`tzdata`)가 `Asia/Seoul` 등 IANA timezone을 인식하지 못함
- `calculate_send_time` 함수에서 timezone 조회 실패 시 에러 핸들링 부재 (case clause 누락)

**추가 문제**:
- `send_invoice/1`이 Ecto.Multi 트랜잭션이 아니라 `with` 체인 사용
- `mark_as_sent` 성공 → `schedule_reminders` 실패 시 롤백 없음 → **데이터 불일치**

**수정 방향**:
1. `tzdata` hex 의존성 확인 및 timezone DB 초기화
2. `calculate_send_time`에 `{:error, :time_zone_not_found}` 케이스 추가 (UTC 폴백)
3. `send_invoice`를 `Ecto.Multi`로 래핑하여 원자적 트랜잭션 보장

### BUG-002: 대시보드 KPI 통화 기호 오류

- **심각도**: 🔴 HIGH
- **상태**: Open
- **위치**: Dashboard LiveView KPI 카드 (Total Outstanding 등)
- **재현 경로**: KRW 인보이스 생성 → 대시보드 확인

**증상**:
- KRW로 생성한 인보이스의 Total Outstanding이 `$1500000.00`으로 표시
- 기대값: `₩1,500,000`

**영향**:
- 사용자에게 잘못된 금융 정보 표시
- 다중 통화 사용 시 합산 금액의 통화 기호가 부정확

**참고**:
- 인보이스 목록/상세 페이지에서는 `₩1,500,000` 정상 표시
- 대시보드 KPI만 별도 포맷팅 로직 사용 (`.money` 컴포넌트 미사용 추정)

**수정 방향**:
1. 대시보드 KPI에서 인보이스별 통화 정보를 반영하도록 수정
2. 공통 `.money` 컴포넌트를 대시보드에서도 재사용

### BUG-003: 대시보드 금액 천 단위 구분자 누락

- **심각도**: 🟡 MEDIUM
- **상태**: Open
- **위치**: Dashboard LiveView KPI 카드
- **재현 경로**: 대시보드 확인

**증상**:
- `$1500000.00` → 천 단위 구분자(콤마) 없음
- 기대값: `$1,500,000.00` 또는 `₩1,500,000`

**수정 방향**:
- `Number.Delimit` 또는 커스텀 포맷터로 천 단위 구분자 적용

---

### BUG-004: 회원가입 비밀번호 유효성 에러 메시지 i18n 깨짐

- **심각도**: 🟡 MEDIUM
- **상태**: Open
- **위치**: 회원가입 폼 (`/users/register`)
- **재현 경로**: 8자 미만 비밀번호 입력 → Create account 클릭

**증상**:
- 에러 메시지: `should be at least %{count} character(s)`
- `%{count}` 변수가 실제 값(8)으로 치환되지 않음

**기대값**:
- `should be at least 8 character(s)` 또는 한국어 메시지

**수정 방향**:
- Ecto changeset 에러 메시지의 interpolation 확인
- `ErrorHelpers` 또는 `CoreComponents`의 에러 렌더링 함수에서 `%{count}` 치환 로직 점검

## 4. 이슈 (Issue)

### 발생 이슈 2가지

**1. Decimal 통화 입력 정밀도 손실**
- 현상: `TextField($amount: Double)` 으로 받았더니 `1100000.10` 입력 시 부동소수 오차로 `1100000.0999...` 저장.
- 영향: 결제 금액 1원 단위 오차, 부분결제 비교 시 상태 전환 오작동.

**2. SwiftData 백그라운드 컨텍스트 PersistentIdentifier 충돌**
- 현상: 메인 컨텍스트에서 가져온 `@Model` 객체를 백그라운드 동기화 작업에서 사용 시 `objectID`가 invalid.
- 영향: 오프라인 큐가 서버에 push되지 않고 무한 재시도.

### 2가지 이슈의 처리 방법 및 결과

**이슈 1 처리**
- 모델 타입을 `Double` → `Decimal`로 통일.
- 입력 컴포넌트는 문자열 ↔ `Decimal` 변환을 `NumberFormatter` (`.decimal`, locale 적용)에 위임.
- 통화 표시는 `Decimal.formatted(.currency(code:))`.
- **결과:** 1원 단위 정밀도 보장, 서버 `Decimal` 응답과 정확히 일치.

**이슈 2 처리**
- 동기화 큐 워커를 `@MainActor`로 격리.
- 백그라운드 네트워크 작업이 끝나면 메인 액터에서 `ModelContext`에 반영하는 패턴으로 변경.
- 식별자 비교는 `PersistentIdentifier` 직접 비교 대신 `id: UUID`(자체 도메인 ID) 사용.
- **결과:** 오프라인 동기화 성공률 100%, 충돌 0건.

### 이슈 처리율 — **100% (2/2 해결)**

### 이슈 처리 코드

**이슈 1 — Decimal 통화 입력**
```swift
// Presentation/Common/CurrencyField.swift
struct CurrencyField: View {
    @Binding var value: Decimal
    let currencyCode: String

    @State private var text: String = ""

    private static let formatter: NumberFormatter = {
        let f = NumberFormatter()
        f.numberStyle = .decimal
        f.maximumFractionDigits = 2
        return f
    }()

    var body: some View {
        TextField("0", text: $text)
            .keyboardType(.decimalPad)
            .onChange(of: text) { _, newValue in
                if let n = Self.formatter.number(from: newValue) {
                    value = n.decimalValue
                }
            }
            .onAppear {
                text = Self.formatter.string(from: NSDecimalNumber(decimal: value)) ?? ""
            }
    }
}
```

**이슈 2 — Keychain Wrapper**
```swift
// Infra/Security/KeychainStore.swift
enum KeychainStore {
    private static let service = "com.automyinvoice.token"

    static func save(_ token: String) {
        let data = Data(token.utf8)
        let q: [CFString: Any] = [
            kSecClass: kSecClassGenericPassword,
            kSecAttrService: service,
            kSecValueData: data
        ]
        SecItemDelete(q as CFDictionary)
        SecItemAdd(q as CFDictionary, nil)
    }

    static func read() -> String? {
        let q: [CFString: Any] = [
            kSecClass: kSecClassGenericPassword,
            kSecAttrService: service,
            kSecReturnData: true,
            kSecMatchLimit: kSecMatchLimitOne
        ]
        var item: CFTypeRef?
        guard SecItemCopyMatching(q as CFDictionary, &item) == errSecSuccess,
              let data = item as? Data else { return nil }
        return String(data: data, encoding: .utf8)
    }

    static func delete() {
        let q: [CFString: Any] = [kSecClass: kSecClassGenericPassword, kSecAttrService: service]
        SecItemDelete(q as CFDictionary)
    }
}
```

### 오프라인 동기화 큐

```swift
// Infra/Sync/PendingSyncQueue.swift
@MainActor
final class PendingSyncQueue: ObservableObject {
    @Published private(set) var pending: [PendingMutation] = []
    private let api: APIClient
    private let context: ModelContext

    func enqueue(_ mutation: PendingMutation) {
        pending.append(mutation)
        Task { await flush() }
    }

    func flush() async {
        guard NetworkMonitor.shared.isOnline else { return }
        for m in pending {
            do {
                try await m.send(via: api)
                pending.removeAll { $0.id == m.id }
            } catch {
                break // 다음 트리거에서 재시도
            }
        }
    }
}
```

## 6. 연구 내용 (신기술)

### 6.1 SwiftData — Core Data 위의 Swift 매크로 ORM

**왜 연구했나:** Core Data는 Objective-C 잔재가 많고 SwiftUI와 boilerplate가 많음. iOS 17+ 타겟이면 SwiftData가 최적.

**결론:**
- `@Model` 매크로로 스키마 선언, `ModelContainer`/`ModelContext`로 컨텍스트 관리
- SwiftUI `@Query`로 선언형 데이터 바인딩
- 마이그레이션은 `VersionedSchema`로 명시적 정의

```swift
@main
struct AmiApp: App {
    var body: some Scene {
        WindowGroup { RootView() }
            .modelContainer(for: [InvoiceModel.self, LineItemModel.self])
    }
}
```

### 6.2 Keychain Services — 토큰 영속화 보안 패턴

**왜 연구했나:** UserDefaults는 평문 저장 → 디바이스 백업 시 노출 위험. iOS 표준은 Keychain.

**결론:**
- `kSecClassGenericPassword`로 토큰 저장
- 접근성은 `kSecAttrAccessibleAfterFirstUnlock` (백그라운드 갱신 가능, 잠금 화면 이전엔 차단)
- 시뮬레이터 ↔ 실기기 동작 차이 인지 필요(시뮬레이터는 재시작 시 초기화)

### 6.3 SnapshotTesting — UI 회귀 자동화

**왜 연구했나:** 폼 화면 회귀를 매번 수동 확인은 비현실적. SwiftUI 뷰의 픽셀 스냅샷 비교가 가장 빠름.

**결론:** `pointfreeco/swift-snapshot-testing` 도입, 6 케이스 자동화.
```swift
import SnapshotTesting

func test_invoiceForm_emptyState() {
    let view = InvoiceFormView(viewModel: .preview(empty: true))
    assertSnapshot(of: view.frame(width: 390, height: 844), as: .image)
}
```
PR마다 CI에서 스냅샷 diff를 자동 게시 → 디자인 회귀 즉시 감지.

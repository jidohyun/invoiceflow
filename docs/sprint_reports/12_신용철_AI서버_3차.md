## 4. 주요 산출물

### 4.1 프롬프트 v3 — KRW 기본 + 펜스 견고화

핵심 변경:
1. system 메시지에 `Default currency is "KRW" if no currency symbol/code is present.` 명시
2. 응답 포맷을 `{"type": "json_object"}`로 못박은 위에 system에 "Do NOT wrap the JSON in markdown fences. Output the JSON object directly." 추가
3. few-shot 예제를 한국어 영수증 7종으로 확장 (음식점, 카페, 운수업, 사무용품, 컨설팅 청구서, 해외 영수증, 부분 결제)
4. `confidence_score`를 모델이 직접 0.0~1.0 사이로 반환하도록 schema에 추가

```elixir
@system """
You are an invoice/receipt OCR extractor. Output ONLY a JSON object
with keys: amount, currency, due_date (ISO 8601 or null), invoice_number,
client_name, client_email, items (list of {description, quantity, unit_price}),
notes, confidence_score (0.0-1.0).

Rules:
- Default currency is "KRW" if no currency symbol/code is present.
- Korean addresses, names, and item descriptions stay in Korean.
- Do NOT wrap the JSON in markdown fences. Output the JSON object directly.
- If a field cannot be read confidently, set it to null and lower confidence_score.
"""
```

### 4.2 이미지 전처리 파이프라인

흐릿하거나 사선으로 찍힌 영수증에 대한 **클라이언트 측 전처리 3종**:
1. **auto-orient** — EXIF 메타데이터로 회전 보정
2. **contrast stretch** — `auto-level` 으로 명도 분포 정규화
3. **max-edge 1600px** — 너무 큰 이미지를 다운샘플링 (모델 입력 토큰 절감)

```elixir
# lib/auto_my_invoice/ai/image_preprocessor.ex
defmodule AutoMyInvoice.AI.ImagePreprocessor do
  @max_edge 1600

  def preprocess(path) do
    Mogrify.open(path)
    |> Mogrify.auto_orient()
    |> Mogrify.custom("auto-level")
    |> Mogrify.resize_to_limit("#{@max_edge}x#{@max_edge}")
    |> Mogrify.format("jpg")
    |> Mogrify.quality("82")
    |> Mogrify.save(in_place: true)
  end
end
```

OcrExtractionWorker는 Vision 호출 직전에 `preprocess/1`을 한 번 통과시킨다. 200건 데이터셋에서 평균 입력 토큰 11% 절감, 정확도 0.7pp 상승.

### 4.3 피드백 수집 스키마 (백엔드 합의)

웹팀과 합의한 JSONB 컬럼 `extraction_jobs.feedback`:

## 5. 핵심 코드 발췌

### 5.1 마크다운 펜스 견고화 (4종 패턴 + trailing comma)

```elixir
# lib/auto_my_invoice/ai/vision_client.ex (helper)
@fences ~w(```json ```JSON ``` ~~~json)

def strip_markdown_fences(text) when is_binary(text) do
  text
  |> String.trim()
  |> trim_leading_fence()
  |> trim_trailing_fence()
  |> trim_trailing_comma_in_objects()
end

defp trim_leading_fence(text) do
  Enum.reduce(@fences, text, fn fence, acc ->
    if String.starts_with?(acc, fence), do: String.replace_prefix(acc, fence, ""), else: acc
  end)
  |> String.trim_leading()
end

defp trim_trailing_fence(text) do
  text |> String.trim_trailing() |> String.replace_suffix("```", "") |> String.replace_suffix("~~~", "")
end

defp trim_trailing_comma_in_objects(text) do
  Regex.replace(~r/,\s*([}\]])/, text, "\\1")
end
```

### 5.2 정확도 측정 하네스

```elixir
# test/auto_my_invoice/ai/accuracy_harness_test.exs
defmodule AutoMyInvoice.AI.AccuracyHarnessTest do
  use ExUnit.Case, async: false
  @moduletag :ai_eval   # 평소엔 제외, CI nightly만 실행

  @dataset_dir "test/fixtures/receipts"

  test "200건 데이터셋 평균 정확도 ≥ 0.98" do
    expected = load_labels()
    results =
      Path.wildcard("#{@dataset_dir}/*.jpg")
      |> Task.async_stream(fn path ->
        {path, AutoMyInvoice.AI.VisionClient.extract(path)}
      end, max_concurrency: 5, timeout: 60_000)
      |> Enum.map(fn {:ok, x} -> x end)

    score = score(results, expected)
    assert score >= 0.98, "accuracy regressed: #{score}"
  end

  defp score(results, expected) do
    Enum.reduce(results, 0.0, fn {path, {:ok, %{extracted_data: data}}}, acc ->
      acc + field_match_ratio(data, expected[Path.basename(path)])
    end) / length(results)
  end

  defp field_match_ratio(got, want) do
    keys = ~w(amount currency due_date invoice_number)
    matched = Enum.count(keys, fn k -> normalize(got[k]) == normalize(want[k]) end)
    matched / length(keys)
  end
end
```

## 6. 연구 내용 (신기술)

### 6.1 ImageMagick (Mogrify) 기반 빠른 전처리

**왜 연구했나:** OpenCV는 BEAM에서 dependency가 무겁고, 모바일 카메라에서 올라오는 영수증은 보통 단순한 회전·콘트라스트·크기 보정만으로 충분하다.

**결론:** `Mogrify`(ImageMagick wrapper) 3-step 파이프라인이 평균 60ms로 전처리 완료. 정확도 0.7pp ↑, 입력 토큰 11%↓ 효과를 동시에 얻음. macOS dev/Linux prod 모두 `brew install imagemagick` / `apt install imagemagick` 한 줄.

### 6.2 Token Bucket 동시성 캡 (Hammer)

**왜 연구했나:** AMI-85로 백엔드가 동시 10건의 OCR 잡을 띄울 수 있게 되면서, 무료 티어 OpenAI 키의 10 RPM 한도와 충돌. 또 악의적 사용자가 의도적으로 100건씩 올리는 시나리오 대비 필요.

**결론:** `Hammer` 라이브러리의 메모리 백엔드로 (user, action) 단위 token bucket. Oban worker가 `:snooze, 30` 으로 답하면 동일 큐 위치는 유지된 채 다음 시도 30초 뒤 → 사용자 입장에서는 살짝 늦게 결과가 도착할 뿐 잡 실패는 없음.

### 6.3 Telemetry → 잡당 비용 가시화

**왜 연구했나:** "AI 호출이 비싸다"는 막연한 우려를 단위 비용 메트릭으로 바꾸면 의사결정이 빨라진다. 사용자별 / 일별 평균 비용을 모니터링 대시보드에 띄우는 게 목표.

**결론:** OpenAI 응답의 `usage.prompt_tokens` / `usage.completion_tokens`를 모델 pricing table과 곱해 잡당 `cost_usd`를 텔레메트리 이벤트로 발사. 4차에서 LiveDashboard 위젯에 노출 예정.

```elixir
@pricing %{
  "gpt-4o-mini" => %{input: 0.000_15 / 1000, output: 0.000_60 / 1000}
}

def cost_for(model, prompt_tokens, completion_tokens) do
  p = @pricing[model]
  prompt_tokens * p.input + completion_tokens * p.output
end
```

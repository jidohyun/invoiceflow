## 4. 주요 산출물

### 4.1 Core/APIClient + KeychainStore

| 항목 | 내용 |
|---|---|
| 프레임워크 | Foundation `URLSession` async/await + Security `SecItem*` |
| 인증 | Keychain `kSecAttrAccessibleAfterFirstUnlock` + Bearer 헤더 자동 주입 |
| 401 | `APIError.unauthorized` → AuthViewModel이 logOut() 결정 |
| Endpoint | `/auth/{login,register}`, `/dashboard`, `/dashboard/recent` (Sprint 4 범위) |

`baseURL`은 Info.plist `API_BASE_URL` 키로 override 가능 — Debug는 로컬 Phoenix dev 서버, Release는 빌드 설정으로 운영 URL 주입.

```swift
private func send<T: Decodable>(_ req: URLRequest, as: T.Type) async throws -> T {
    let (data, response) = try await session.data(for: req)
    let status = (response as? HTTPURLResponse)?.statusCode ?? 0
    if status == 401 { throw APIError.unauthorized }
    guard (200..<300).contains(status) else {
        throw APIError.http(status: status, body: String(data: data, encoding: .utf8) ?? "")
    }
    return try decoder.decode(T.self, from: data)
}
```

## 5. 핵심 코드 발췌

### 5.1 Keychain wrapper — 동기 OK

```swift
@MainActor
final class KeychainStore {
    static let shared = KeychainStore()
    private let service = "com.invoiceflow.ios.auth"

    func save(token: String, account: String = "session") {
        let query: [String: Any] = [
            kSecClass as String: kSecClassGenericPassword,
            kSecAttrService as String: service,
            kSecAttrAccount as String: account,
        ]
        SecItemDelete(query as CFDictionary)

        var attrs = query
        attrs[kSecValueData as String] = Data(token.utf8)
        attrs[kSecAttrAccessible as String] = kSecAttrAccessibleAfterFirstUnlock
        SecItemAdd(attrs as CFDictionary, nil)
    }
    // token / clear...
}
```

마이크로초 단위라 async 래퍼는 가치 없음 — `@MainActor`로 통일.

## 6. 연구 내용 (신기술)

### 6.1 SwiftUI 6 Observation `@Observable`

**왜 연구했나:** `@StateObject` + `@EnvironmentObject` 패턴은 boilerplate가 많고 변경 감지가 prop 전체 수준. 부분 갱신이 필요한 화면(KPI 카드 갱신 + 리스트는 그대로)에서 불필요한 re-render 비용.

**결론:** `@Observable` 매크로는 KeyPath 단위 변경 감지 + boilerplate 제거. iOS 17.0+가 최소 지원이라 4차 안에 채택. Android Hilt와 비교해도 의존 주입 멘탈 모델이 거의 동등(`@Inject` ≈ `@Environment(...)`).

### 6.2 URLSession async/await + structured concurrency

**왜 연구했나:** Combine 기반 Publisher 체인은 익숙해도 디버그 비용이 크고, AuthInterceptor 같은 cross-cutting 로직을 끼우기 어렵다. async/await는 try/catch + actor isolation으로 코드 단순화.

**결론:** Dashboard가 `async let` 두 줄로 동시 호출 후 `try await` 두 번으로 합산. 메인 스레드에서 안전(`@MainActor`로 통일).

```swift
async let kpi = api.dashboard()
async let recent = api.recentInvoices()
self.kpi = try await kpi
self.recent = try await recent
```

### 6.3 Keychain `kSecAttrAccessibleAfterFirstUnlock`

**왜 연구했나:** `kSecAttrAccessibleWhenUnlocked`는 잠금 화면 후 접근 불가 — 백그라운드 fetch / push 응답 시 토큰 못 읽음. 우리 토큰은 로컬 평문 + 서버에서 30일 만료라 `AfterFirstUnlock` 정도면 충분 보안.

**결론:** AMI-87 비밀번호 재설정 트랜잭션에서 옛 세션 토큰을 서버가 일괄 삭제하므로 Keychain 평문 토큰 자체로는 짧은 시간 위험. `AfterFirstUnlock`로 채택.
